# spec-planning

## 目的

分析使用者提供的規範文件，產出開發規劃，並取得使用者確認後才進入開發。

## 適用範圍

- 需跨文件整合規格來源
- 開發功能前的規劃階段

## 必要輸入

- **所有規範參考文件**（由使用者提供所有相關文件的連結或內容）

## MCP 取得 Confluence 文件

當使用者提供 Confluence 連結時，使用 mcp 工具取得頁面內容。

## 流程

1. 請使用者提供所有規範參考文件
2. 分析並整合使用者提供的文件
   - 識別功能需求與規格
   - 標註不明確或衝突處
3. 產出開發規劃
   - 功能拆分與結構
   - 待確認事項清單
4. 提供開發規劃讓使用者確認
   - 未確認前不得進入 spec 開發

## 輸出

- 開發規劃摘要
- 待確認事項清單

## 注意事項

- 嚴格依據使用者提供的文件，禁止自行假設
- 規格不完整時先確認再進行 spec 開發
