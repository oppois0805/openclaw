# 表單驗證規則分析 Prompt

## 使用情境

當需要從規格文件中提取完整的表單驗證規則時使用 mcp 取得相關文件，網址只是參考不是直接去讀取。

---

## Prompt 模板

```markdown
據以下規格文件，分析並整理出完整的表單驗證項目清單：

### 規格文件來源

1. **系統設計規則文件**：[提供連結或文件名稱]
2. **功能規格文件**：[提供連結或文件名稱]

### 分析要求

#### 一、從系統設計規則提取通用元件驗證規範

請識別以下元件的驗證規則：

- Input（輸入框）：maxLength、required、格式驗證
- InputNumber（數字輸入）：min、max、預設值
- Dropdown（下拉選單）：required
- Checkbox（勾選框）：至少勾選一項
- Calendar/DatePicker（日期選擇）：日期範圍、過期驗證

#### 二、從功能規格文件提取特定驗證項目

請針對每個表單/Dialog 整理：

- 欄位名稱
- 欄位類型
- 驗證規則（required、maxLength、min/max、pattern 等）
- i18n Key（錯誤訊息對應）
- 說明

#### 三、識別業務邏輯驗證（非同步）

- 唯一性檢查（如：名稱不可重複）
- 衝突檢查（如：資源已被使用）
- 跨欄位驗證（如：結束日期 >= 開始日期）

#### 四、輸出格式要求

##### 通用元件驗證規範表

| 元件類型 | 驗證規則 | 說明 |
| -------- | -------- | ---- |

##### 功能特定驗證項目表（每個 Dialog/Form 一張表）

| 欄位 | 類型 | 驗證規則 | i18n Key | 說明 |
| ---- | ---- | -------- | -------- | ---- |

##### 非同步驗證項目表

| 驗證項目 | 類型 | 觸發時機 | i18n Key | 說明 |
| -------- | ---- | -------- | -------- | ---- |

##### 驗證器實作清單

- 同步驗證器（shared/validators/sync.validators.ts）
- 非同步驗證器（shared/validators/async.validators.ts）

##### i18n 錯誤訊息對應表

| Key | EN  | ZH-TW |
| --- | --- | ----- |

##### 待確認事項（規格不明確處）

| 欄位 | 待確認項目 | 建議預設值 |
| ---- | ---------- | ---------- |

### 注意事項

- 嚴格依據規格文件，禁止自行假設
- 規格中沒有明確定義的項目，請查找系統設計規則
- 仍找不到的項目列入「待確認事項」
```

## 預期輸出結構

分析完成後應產出：

1. ✅ 通用元件驗證規範（來自系統設計規則）
2. ✅ 功能特定驗證項目（每個表單/Dialog）
3. ✅ 非同步驗證項目（業務邏輯）
4. ✅ 驗證器實作清單
5. ✅ i18n 錯誤訊息對應
6. ⚠️ 待確認事項（需與 PM 確認）
